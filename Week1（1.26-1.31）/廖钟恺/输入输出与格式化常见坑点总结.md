# 输入输出与格式化常见坑点总结

## 1. 空格处理相关

### 坑点1：输出格式中多余的空格
- **问题**：在输出多个数值或字符串时，容易在末尾多输出一个空格
- **解决方案**：
①元素前加空格：使用标志位控制空格输出，仅在非首个元素前添加空格（L1-020）

  ```cpp
  bool first = true;
  for (auto item : items) {
      if (!first) cout << " ";
      cout << item;
      first = false;
  }
  ```
②元素后加空格：在每个元素输出后添加空格，输出时去除最后一个空格（L1-007）
  ```cpp
  string output;
  for (auto item : items) {
      output += item + " ";
  }
  output.pop_back(); // 去除最后一个空格
  cout << output;
  ```
### 坑点2：输入包含空格的字符串
- **问题**：使用cin读取字符串时会自动跳过空格和换行符（L1-011）
- **解决方案**：
  - 使用getline()读取整行
  - 使用cin.getline()或cin.ignore()处理换行符残留

## 2. 数字精度相关

### 坑点3：浮点数精度控制
- **问题**：浮点数输出时精度不符合题目要求
- **解决方案**：使用setprecision()和fixed控制输出精度
  ```cpp
  #include <iomanip>
  cout << fixed << setprecision(2) << number; // 保留2位小数
  ```

### 坑点4：整数溢出
- **问题**：使用int类型处理超出范围的数据
- **解决方案**：根据数据规模选择合适的数据类型（int -> long long,使用string存储大数等）

## 3. 数据结构选择相关

### 坑点5：容器选择不当
- **问题**：对查找操作频繁的场景使用vector而非unordered_set
- **示例**：L1-020 帅到没朋友中，需要快速判断某ID是否在朋友圈中
- **解决方案**：根据操作特点选择合适的STL容器

## 总结
在PTA刷题过程中，格式问题是导致WA的常见原因。编写代码时需要特别关注：
1. 严格按照题目要求的格式输出
2. 注意空行、空格、换行符的位置
3. 正确处理边界情况
4. 选择合适的数据结构和算法
